05

Root Cause and Fix Guidance

Vulnerable Flow

The current flow reads byteBufMsg.array(), calls safeRelease, then encrypts appDataBytes.length. This combines a strict Use-After-Free with full-capacity over-read.

Result: encrypted output includes stale pooled bytes beyond valid payload boundaries.

Remediation Direction

In TlsPskHandler.write, compute length = readableBytes(), respect arrayOffset + readerIndex when using array-backed buffers, and release only in a finally block after encryption/output write.

Remove helper logic that returns array references after release.
